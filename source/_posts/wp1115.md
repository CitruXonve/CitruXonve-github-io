class Pt{
    int x,y,n;
    Pt(int _x,int _y,int _n){
        x=_x;y=_y;n=_n;
    }
    Pt(int hash,int _n){
        n=_n;
        y=hash%n;
        x=hash/n;
    }
    public int hashCode(){
        return x*n+y;
    }
}

折腾了过长的时间，原因：   
这不是通常意义上的可随意匹配的二分图。   
既然是二分图，就有把节点分为两类的依据。   
由题中给定的结合规则，一定是一个(x+y)为偶数的点与一个(x+y)为奇数的点相匹配，且是相邻点的匹配。   
所以会将两类点分别加入两个表中，并进行hash操作节约存储空间。   
“奇”的点总是不少于“偶”的点？
